# 设计演讲评分要求

| 评分项目                                                     | 满分 | 得分 |
| ------------------------------------------------------------ | ---: | ---: |
| 是否准备了相关的演示材料？                                   |    5 |      |
| 内容完整性如何？                                             |    5 |      |
| 展示效果如何？                                               |   10 |      |
| 需求分析与系统设计是否使用了 UML 工具？                      |   10 |      |
| 需求分析是否清晰？是否可以移交给另一小组实现而依然理解正确？ |   10 |      |
| 是否考虑架构设计？                                           |   10 |      |
| 是否考虑接口设计？                                           |   10 |      |
| 是否考虑构件设计？                                           |   10 |      |
| 是否使用了设计模板（Design Pattern）？                       |   10 |      |
| 是否有质量保证措施？                                         |    5 |      |
| 是否每个组员均有任务分配？任务分配是否合理？                 |    5 |      |
| 是否经常召开项目会议，并有会议记录？                         |    5 |      |
| 是否对后期工作有安排？                                       |    5 |      |
| 合计                                                         |  100 |      |

## 其他建议

1. 展示时间约 25 分钟
2. 每个小组在介绍当前进度时，用截图或视频来做简单展示
