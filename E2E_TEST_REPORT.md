## 修复记录

### 1. 测试凭据错误修复

**问题**: 测试使用错误凭据 `Admin123!`，实际密码为 `admin123`

**修复方案**:

- 修改 `frontend/tests/e2e/fixtures/auth.ts` 中的默认密码为 `admin123`
- 修改 `frontend/tests/e2e/auth.spec.ts` 中的测试密码为 `admin123`

**验证结果**: ✅ 已修复

### 2. 路由错误修复

**问题**: 测试使用 `/users` 路由，实际路由为 `/info/users`

**修复方案**:

- 修改所有测试用例使用正确的路由 `/info/users`

**验证结果**: ✅ 已修复

### 3. 登录重定向错误修复

**问题**: 测试期望登录后重定向到 `/dashboard`，实际重定向到 `/`（首页即 Dashboard）

**修复方案**:

- 修改 auth fixture 和 auth.spec.ts 中的重定向期望为 `/`

**验证结果**: ✅ 已修复

### 4. 用户列表元素选择器修复

**问题**: 测试使用 `.ant-input-search input` 选择器，实际组件使用 `.ant-input-search`

**修复方案**:

- 修改测试用例使用更精确的选择器 `.ant-input-search`

**验证结果**: ✅ 已修复

### 5. 登录失败错误消息显示问题

**问题**: 登录失败时应该显示错误消息，但测试超时

**修复方案**:

- 修改测试用例使用更宽松的选择器和更长的超时时间
- 使用 `page.waitForSelector` 代替 `expect.toBeVisible`

**验证结果**: ❌ 仍有问题，需要进一步调查 Ant Design message 的显示时机

可能的原因:

1. 后端返回的错误消息格式可能与前端预期不同
2. Ant Design message 组件的 DOM 结构可能在测试环境中不同
3. 错误消息显示的时机比预期更晚

需要进一步调查:

- 检查后端返回的实际错误消息格式
- 检查前端如何解析和显示错误消息
- 检查 Ant Design message 在测试环境中的实际 DOM 结构

### 6. Auth Fixture 登录失败

**问题**: 使用 authenticatedUser fixture 的测试全部失败，无法完成登录

**修复方案**:

- 已修复密码为 `admin123`
- 已修复重定向为 `/`

**验证结果**: ❌ 仍然失败，可能是以下原因:

1. 登录流程本身有问题（如表单提交后没有正确处理响应）
2. 后端登录 API 有问题
3. 前端登录逻辑有bug

需要进一步调查:

- 检查登录表单提交是否正确触发了 API 调用
- 检查后端是否正确返回了登录响应
- 检查前端是否正确处理了登录响应并更新了状态
- 检查路由跳转是否正确触发
