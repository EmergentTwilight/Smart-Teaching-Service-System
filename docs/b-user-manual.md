## 3. 启动方式

在项目根目录执行：

```bash
docker-compose up -d server web postgres redis adminer
```

常用访问地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:3000`
- 健康检查：`http://localhost:3000/api/health`
- Adminer：`http://localhost:8080`

启动确认：

- 前端页面能打开
- 健康检查返回 `200`

## 5. 教室管理

入口：

- 左侧菜单进入“教室管理”

常见操作：

1. 在列表页按关键字、校区、类型筛选
2. 点击“新增教室”
3. 填写校区、教学楼、教室号、容量、类型、状态
4. 在“设备信息”中填写投影仪、空调、麦克风、电脑数量
5. 点击“提交”

编辑教室：

1. 在列表中点击“编辑”
2. 修改信息
3. 点击“提交”

说明：

- 设备信息现在按嵌套 `equipment` 保存
- “维护中”状态可正常保存

## 6. 手动排课

入口：

- 左侧菜单进入“手动排课”

新增排课：

1. 点击“手动排课”
2. 输入 `CourseOffering ID`
3. 选择星期、开始周次、结束周次、开始节次、结束节次
4. 等待系统自动加载可用教室
5. 选择教室
6. 点击“校验并保存”

编辑排课：

1. 在列表中点击“编辑”
2. 修改时间或教室
3. 点击“校验并保存”

删除排课：

1. 在列表中点击删除按钮
2. 确认删除

说明：

- 保存前会先做冲突校验
- 若有冲突，页面会显示冲突信息，不会直接落库

## 7. 课表查看

入口：

- 左侧菜单进入“课表查看”

支持的查看方式：

- 综合视图
- 按教室查看
- 按课程查看

使用方法：

1. 选择视图模式
2. 按页面要求选择学期、教室或课程
3. 页面会自动加载课表数据

导出说明：

- 当前只支持导出 `CSV`
- 页面导出按钮会下载 `.csv` 文件

## 8. 规则管理

入口：

- 自动排课页面中的“约束规则管理”

新增规则：

1. 点击“新增规则”
2. 选择目标类型：教师 / 课程
3. 输入目标 ID
4. 配置硬约束和软约束
5. 点击“保存”

规则说明：

- `requiredRoomType` 现在只接受大写枚举：
  - `LECTURE`
  - `LAB`
  - `COMPUTER`
  - `MULTIMEDIA`

## 9. 自动排课

入口：

- 左侧菜单进入“自动排课”

操作步骤：

1. 先确认学期已选中
2. 检查规则列表
3. 点击“开始自动排课”
4. 等待任务进入预览阶段
5. 查看成功率和失败记录
6. 点击“确认无误，应用落库”

完成后：

- 可到“课表查看”或“手动排课”页面核对结果

## 10. 已知说明

- 导出目前只支持 `CSV`
- 按课程查看的导出默认按全局导出，不直接绑定单门课程
- 自动排课仍是当前分支实现，不代表最终业务规则全部完成
